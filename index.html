workflow-agent

A ticket-lifecycle workflow system for AI coding agents. Structured project management your agent can read, write, and follow autonomously.

init→ breakdown→ plan→ review→ implement→ review→ complete

Why

AI agents lose track of larger projects. They forget context between sessions, skip steps, and can't resume interrupted work. workflow-agent gives your agent a filesystem-based ticket system — markdown files moving between status folders — that persists across sessions and requires no external tools.

Skills

wf-new-init

Bootstrap a new initiative with folder structure and discovery prompts.

"new init", "create initiative"

wf-breakdown

Interview-driven breakdown of scope into independently deployable tickets.

"breakdown", "create tickets"

wf-plan

Explore codebase and write a directional implementation plan into the ticket.

"plan NNN"

wf-plan-review

Review plan for completeness, risks, and feasibility before coding.

"review plan NNN"

wf-implement

Execute the plan step-by-step. Write code, run tests, update ticket.

"implement NNN"

wf-impl-review

Review code changes against plan and acceptance criteria.

"review impl NNN"

wf-complete

Verify all AC met, archive ticket, update progress.

"done NNN", "complete NNN"

wf-manager

General ticket CRUD: create, edit, move, show kanban.

"new ticket", "show kanban"

wf-status

Read-only progress overview across all initiatives.

"status", "progress"

wf-how-to

Explains the workflow system to new users.

"how does workflow work"

Install

git clone https://github.com/YOUR_USER/workflow-agent.git cd workflow-agent ./install.sh

Symlinks all skills to ~/.agents/skills/. Use --force to overwrite existing.

Compatibility

Works with any AI agent that can read and write files.

OpenCode Cursor Windsurf Claude Code Aider Any LLM + filesystem
