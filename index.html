workflow-agent

A ticket-lifecycle workflow system for AI coding agents. Structured project management your agent can read, write, and follow autonomously.

Core Flow

Step 1

wf-plan

Explore the codebase and write a directional implementation plan for the ticket.

→
Step 2

wf-plan-review

Review the plan for completeness, risks, and feasibility before any code is written.

→
Step 3

wf-implement

Execute the plan step-by-step — write code, run tests, update ticket progress.

→
Step 4

wf-impl-review

Review actual code changes against the plan and acceptance criteria.

The Full Lifecycle

From initial concept to completed features, workflow-agent manages the entire process end-to-end.

Init

wf-new-init

Prepare for a new initiative. This scaffolds the necessary folder structure for your project inputs and context.

Discover

wf-discover

Structured interview to flesh out SPEC.md. Agent asks targeted questions, identifies gaps, and writes a complete specification ready for breakdown.

Plan

wf-breakdown

Based on your SPEC.md, this skill automatically splits the scope and generates a set of manageable, independently deployable tickets.

Action

🔁 Execute Core Flow

For each ticket, loop through the Core Flow (above). wf-plan, review the plan, wf-implement, and finally review the implementation.

Finish

wf-complete

Once you are happy with a ticket and it meets all acceptance criteria, mark it complete. It automatically moves to the 'done' folder.

Track

wf-status

Use this anytime to get a quick update on progress, check what is currently in flight, and see exactly what needs to be done next.

Why

AI agents lose track of larger projects. They forget context between sessions, skip steps, and can't resume interrupted work. workflow-agent gives your agent a filesystem-based ticket system — markdown files moving between status folders — that persists across sessions and requires no external tools.
